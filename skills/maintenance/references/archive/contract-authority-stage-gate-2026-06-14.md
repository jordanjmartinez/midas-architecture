# Contract authority stage-gate lesson — 2026-06-14

Archived session note. This is preserved context, not active routing law.

Reusable lesson distilled into `SKILL.md`:

- Before patching Midas rules, contracts, references, templates, schemas, docs, eval requirements, or command-local instructions, perform a Contract Authority Check.
- Make the proposed authority layer visible before editing.
- If behavior applies across commands, prefer a global `rules/` home plus short pointers.
- If content is one-off or historical, archive it instead of promoting it to active routing law.
- Preserve stage boundaries: proposal → approval → execution → verification → stop → next approval.
